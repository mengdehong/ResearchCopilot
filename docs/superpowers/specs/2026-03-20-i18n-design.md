# 前端 i18n 设计：轻量中英双语

> 零依赖、类型安全的 React Context i18n 方案，对齐 `react-i18next` API 以降低未来迁移成本。

## 约束

- **语言范围**：仅中文 + 英文
- **翻译范围**：UI 静态文本 + 学科枚举。不翻译：placeholder、Agent 生成内容、日期格式（由 `Intl` 自动处理）
- **语言检测**：`localStorage` → `navigator.language` → 默认 `'en'`
- **手动切换**：Settings 页 Preferences 区块首位

## 架构

```
src/i18n/
├── types.ts              # Locale 类型、FlatKeys 工具类型
├── locales/
│   ├── en.ts             # 英文字典（类型源 ≡ 合约）
│   └── zh.ts             # 中文字典（满足 TranslationDict）
├── context.tsx           # LocaleProvider + LocaleContext
└── useTranslation.ts     # 唯一对外 hook：{ t, locale, setLocale }
```

### 数据流

```
detectLocale() → LocaleProvider(state) → useTranslation() → t('key')
                        ↑                                   
               setLocale() + localStorage.setItem()
```

## 核心类型

```typescript
// i18n/types.ts
export type Locale = 'en' | 'zh'

// 递归展平嵌套对象 key 为 dot-path 联合类型
// { common: { save: string } } → 'common.save'
export type FlatKeys<T, Prefix extends string = ''> =
  T extends string
    ? Prefix
    : { [K in keyof T & string]:
        FlatKeys<T[K], Prefix extends '' ? K : `${Prefix}.${K}`>
      }[keyof T & string]
```

## Provider + Hook 实现

```typescript
// i18n/context.tsx
import { createContext, useState, useCallback, type ReactNode } from 'react'
import { en } from './locales/en'
import { zh } from './locales/zh'
import type { Locale, FlatKeys } from './types'
import type { TranslationDict } from './locales/en'

const dictionaries: Record<Locale, TranslationDict> = { en, zh }

function detectLocale(): Locale {
  const stored = localStorage.getItem('locale')
  if (stored === 'en' || stored === 'zh') return stored
  return navigator.language.startsWith('zh') ? 'zh' : 'en'
}

// 运行时按 dot-path 取值
function resolve(dict: TranslationDict, key: string, params?: Record<string, string>): string {
  const value = key.split('.').reduce<unknown>((obj, k) => (obj as Record<string, unknown>)?.[k], dict)
  if (typeof value !== 'string') return key  // fallback 显示 key 本身
  if (!params) return value
  return value.replace(/\{\{(\w+)\}\}/g, (_, k: string) => params[k] ?? `{{${k}}}`)
}

interface LocaleContextValue {
  locale: Locale
  setLocale: (locale: Locale) => void
  t: (key: FlatKeys<TranslationDict>, params?: Record<string, string>) => string
}

export const LocaleContext = createContext<LocaleContextValue>(null!)

export function LocaleProvider({ children }: { children: ReactNode }) {
  const [locale, setLocaleState] = useState<Locale>(detectLocale)

  const setLocale = useCallback((next: Locale) => {
    setLocaleState(next)
    localStorage.setItem('locale', next)
    document.documentElement.lang = next
  }, [])

  const t = useCallback(
    (key: FlatKeys<TranslationDict>, params?: Record<string, string>) =>
      resolve(dictionaries[locale], key, params),
    [locale],
  )

  return (
    <LocaleContext value={{ locale, setLocale, t }}>
      {children}
    </LocaleContext>
  )
}
```

```typescript
// i18n/useTranslation.ts
import { use } from 'react'
import { LocaleContext } from './context'

export function useTranslation() {
  return use(LocaleContext)
}
```

## Settings 页语言切换

在 Preferences section 中，**Language 放在首位**，Default Discipline 紧随其后：

```tsx
<section className="settings-section card">
  <h2>{t('settings.preferences')}</h2>

  {/* Language — 首位 */}
  <div className="settings-field">
    <label htmlFor="language">{t('settings.language')}</label>
    <select
      id="language"
      className="settings-input"
      value={locale}
      onChange={(e) => setLocale(e.target.value as Locale)}
    >
      <option value="en">English</option>
      <option value="zh">中文</option>
    </select>
  </div>

  {/* Default Discipline */}
  <div className="settings-field">
    <label htmlFor="discipline">{t('settings.defaultDiscipline')}</label>
    <select id="discipline" className="settings-input">
      {(['computer_science','biology','physics','mathematics','chemistry','other'] as const).map(d => (
        <option key={d} value={d}>{t(`discipline.${d}`)}</option>
      ))}
    </select>
  </div>
</section>
```

## 迁移友好性保证

| 设计选择                                                | 迁移到 `react-i18next` 时的改动       |
| ------------------------------------------------------- | ------------------------------------- |
| `useTranslation()` hook 返回 `{ t, locale, setLocale }` | 只需改 hook 内部实现                  |
| `t('namespace.key')` dot-path 调用                      | i18next 原生支持                      |
| `t('key', { name: 'val' })` 插值                        | 改 `{{}}` 为 i18next 默认格式（相同） |
| 翻译文件为独立 TS 模块                                  | 转为 JSON 即可被 i18next backend 加载 |
| 组件层零直接依赖 Provider 实现                          | 组件代码零改动                        |
